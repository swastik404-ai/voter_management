Media Management System API Documentation
=======================================

Base URL: http://127.0.0.1:8000/

Authentication
-------------
All API endpoints require authentication using Token Authentication.
Include the token in the header:
Authorization: Token <your-token>

API Endpoints
------------

1. Media List & Create
----------------------
Endpoint: /api/media/
Methods: GET, POST
Description: List all media items or create a new one

GET Response Format:
{
    "count": <total_count>,
    "next": <next_page_url>,
    "previous": <previous_page_url>,
    "results": [
        {
            "id": <int>,
            "title": <string>,
            "slug": <string>,
            "media_type": <string>,
            "media_url": <string>,
            "category": <string>,
            "status": <string>,
            "created_at": <datetime>,
            "author_name": <string>
        }
    ]
}

POST Request Format:
{
    "title": <string>,
    "media_type": "image" | "video",
    "image": <file> | null,
    "video_link": <string> | null,
    "caption": <string>,
    "category": "event" | "campaign" | "interview" | "behind_scenes",
    "tags": <string>,
    "status": "draft" | "published" | "archived"
}

2. Media Detail
--------------
Endpoint: /api/media/<pk>/
Methods: GET, PUT, PATCH, DELETE
Description: Retrieve, update, or delete a specific media item

Response Format:
{
    "id": <int>,
    "title": <string>,
    "slug": <string>,
    "author": <int>,
    "author_name": <string>,
    "media_type": <string>,
    "image": <string>,
    "video_link": <string>,
    "caption": <string>,
    "category": <string>,
    "tags": <string>,
    "status": <string>,
    "created_at": <datetime>,
    "updated_at": <datetime>,
    "published_date": <datetime>,
    "created_by": <string>,
    "created_time": <datetime>
}

3. Archive Media
---------------
Endpoint: /api/media/<pk>/archive/
Method: POST
Description: Archive a specific media item

Response Format:
{
    "status": "success",
    "message": "Media archived successfully",
    "timestamp": <datetime>
}

4. Preview Media
---------------
Endpoint: /admin/Media_Management/media/<pk>/preview/
Method: GET
Description: Get preview of a specific media item

Response: Renders HTML template with media details

5. Media Filters
---------------
Endpoint: /api/media/
Method: GET
Description: Filter media items

Query Parameters:
- status: Filter by status (draft, published, archived)
- media_type: Filter by type (image, video)
- category: Filter by category (event, campaign, interview, behind_scenes)

Example: /api/media/?status=published&media_type=image

Admin API Endpoints
-----------------

1. Admin Media List
------------------
Endpoint: /admin/Media_Management/media/
Method: GET
Description: Display admin interface for media management

2. Admin Media Create
--------------------
Endpoint: /admin/Media_Management/media/add/
Method: GET, POST
Description: Admin interface for creating new media items

3. Admin Media Edit
------------------
Endpoint: /admin/Media_Management/media/<pk>/change/
Method: GET, POST
Description: Admin interface for editing media items

4. Admin Media Archive
---------------------
Endpoint: /admin/Media_Management/media/<pk>/archive/
Method: POST
Description: Admin interface for archiving media items

Response Format:
{
    "status": "success",
    "message": "Media archived successfully",
    "timestamp": <datetime>
}

Error Responses
--------------
All endpoints may return the following error responses:

400 Bad Request:
{
    "status": "error",
    "message": <error_description>
}

401 Unauthorized:
{
    "detail": "Authentication credentials were not provided."
}

403 Forbidden:
{
    "detail": "You do not have permission to perform this action."
}

404 Not Found:
{
    "detail": "Not found."
}

Media File Handling
------------------
- Images are stored in: /media/media_images/
- Supported image formats: JPEG, PNG, GIF
- Maximum file size: 5MB
- Video links support: YouTube and Vimeo URLs

Filtering Examples
----------------
1. Get all published images:
   GET /api/media/?status=published&media_type=image

2. Get all archived videos:
   GET /api/media/?status=archived&media_type=video

3. Get all event photos:
   GET /api/media/?category=event&media_type=image

4. Get all campaign videos:
   GET /api/media/?category=campaign&media_type=video

Note: All timestamps are in UTC format (YYYY-MM-DD HH:MM:SS)
